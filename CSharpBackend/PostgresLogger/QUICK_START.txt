========================================
CEREVEATE DATABASE TRENDS - QUICK START
========================================

CONFIGURATION FILE LOCATION:
----------------------------
PostgresLogger\config\app_config.json

This file controls:
- Database connection (host, port, user, password)
- Parquet source directory (D:\OpcLogs\Data)
- Tag mappings (which parquet columns to import)
- Auto-discovery settings
- Web UI settings (port 8001, refresh intervals)

STARTING THE APPLICATION:
-------------------------
1. START_ALL.bat - Starts both web server AND background importer
   OR
2. start_server.bat - Starts only web server (port 8001)
3. start_importer.bat - Starts only background importer

WEB UI ACCESS:
--------------
http://localhost:8001

TABS IN WEB UI:
---------------
1. Live Trends - Real-time data with auto-refresh (1s to 1min intervals)
2. Historical Trends - Query past data with date picker
3. Tag Configuration - Add/Edit tag mappings, auto-discover from parquet
4. Statistics - Database stats (total records, tags, date ranges)

HOW TAG MAPPING WORKS:
----------------------
1. Parquet files in D:\OpcLogs\Data have columns (e.g., "Random.Real4")
2. Each column needs a mapping in config\app_config.json
3. Mapping includes: Plant, Asset, Subsystem, Unit, Description
4. You can:
   - Manually edit config\app_config.json
   - Use "Auto-Discover" button in web UI
   - Click "Add New Tag" in web UI
   - Click "Edit" button next to any tag

BACKGROUND IMPORTER:
--------------------
- Monitors D:\OpcLogs\Data for new parquet files
- Automatically imports data to PostgreSQL
- Creates file_imports table to track imported files
- Runs independently from web server

TROUBLESHOOTING:
----------------
1. "Edit" button not working:
   - Refresh browser (Ctrl+F5)
   - Check browser console for errors

2. No data showing:
   - Check if background importer is running
   - Verify parquet files exist in D:\OpcLogs\Data
   - Check tag mappings are enabled in config

3. Config changes not applying:
   - Stop and restart the web server
   - Config is loaded at startup

4. Port 8001 already in use:
   - Change port in config\app_config.json under "web_ui" -> "port"
   - Restart server

LINKING FROM OPC APPLICATION:
------------------------------
Add this to your OPC DA app's Index.cshtml:

<li><a href="http://localhost:8001" target="_blank">Database Trends</a></li>

Or in Pages\Index.cshtml, add a tab that opens localhost:8001 in iframe.
