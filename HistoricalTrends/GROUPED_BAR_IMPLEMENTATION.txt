WHAT WAS IMPLEMENTED
================================================================================

1. ✅ SIMPLIFIED CONFIGURATION (trends-config.json)

   - Added simple flags: EnableAutoScaling, EnableAutoDetection

2. ✅ SMART AUTO-DETECTION (bi_analytics.js)
   - Auto-detects ALL numeric tags from loaded data
   - Smart limiting: Shows ALL if ≤10 tags, otherwise shows 6 tags
   - No manual tag selection needed

3. ✅ DYNAMIC SAMPLING (bi_analytics.js)
   - Auto-calculates optimal sample size based on data volume:
     * 100-1K rows: Use all data
     * 1K-10K rows: Sample to 5K
     * 10K-100K rows: Sample to 10K
     * 100K-1M rows: Sample to 50K
     * 1M+ rows: Sample to 100K
   - Handles 100 to 10 MILLION+ rows automatically

4. ✅ AUTOMATIC CALCULATIONS
   - Design/Target: max × 1.05 (5% above maximum)
   - Last Period: Value at 75th percentile
   - Current: Latest value in dataset

DATA FLOW
================================================================================

Main UI (trends.html)
   ↓
Load Data → currentData array (3,269 rows example)
   ↓
Click Plant Analytics → mode = 'bianalytics'
   ↓
trends.js calls: BIAnalytics.initialize(currentData, selectedTags)
   ↓
bi_analytics.js:
   1. Auto-samples data (if needed)
   2. Loads minimal config (EnableAutoDetection: true)
   3. Renders tag selectors
   ↓
Click Grouped Bar → BIAnalytics.renderGroupedBarChart(container)
   ↓
Auto-detection:
   1. Filters timestamp field
   2. Checks typeof === 'number'
   3. Validates !isNaN(value)
   4. Smart limits to display count
   ↓
Render chart with Plotly:
   - Blue bars: Design/Target
   - Yellow bars: Last Period
   - Green bars: Current
