=============================================================================
                        TECHNICAL DETAILS
=============================================================================

THREADING MODEL:
----------------
- Main Flask thread handles requests
- Prediction tasks spawn daemon threads
- Each task gets unique ID
- Frontend polls /api/prediction/status/{task_id}
- No blocking, no freezing

PREDICTION ALGORITHM:
---------------------
1. Identify missing segments in data
2. Extract context (100 points before/after gap)
3. Run selected ML model(s) on context
4. Generate predictions for gap
5. Calculate confidence score
6. Return results with metadata

DATA STRUCTURE:
---------------
prediction_cache.parquet:
┌─────────────┬─────────────────┬───────────────┬─────────┬───────┐
│ Timestamp   │ PredictedValue  │ OriginalValue │ Model   │ TagId │
├─────────────┼─────────────────┼───────────────┼─────────┼───────┤
│ 2025-11-20  │ 245.67         │ NULL          │ fft     │ LOAD  │
│ 08:15:00    │                │               │         │       │
└─────────────┴─────────────────┴───────────────┴─────────┴───────┘

PERFORMANCE:
------------
- Small gaps (<100 pts): 2-5 seconds
- Medium gaps (100-1000 pts): 5-15 seconds
- Large gaps (1000+ pts): 15-30 seconds
- Multiple models: Add proportionally
- System remains responsive throughout
